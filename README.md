## Project Purpose

TITAN ORACLE is not a basic crypto signal app, portfolio tracker, alert screen, or casual trading bot.

The project is being developed as an institutional-style crypto intelligence cockpit that combines:

- Oracle Feed market intelligence
- Signal Pro validation and decision support
- TITAN Δ INSIGHT compact decision briefing
- Setup Signal configuration
- Accept Signal mission activation preview
- Live Radar tactical monitoring
- Mission Center active trade supervision
- Stats Hub outcome intelligence
- Trial-mode diagnosis
- API-cost visibility
- Future guarded AUTO-TRADING, TITAN AI COPILOT EXECUTION, and TITAN AI AUTO PILOT simulation

The system must assist, validate, explain, monitor, diagnose, and support trading decisions. It must not claim guaranteed profit, guaranteed prediction accuracy, or guaranteed trade outcomes.

Final trading decisions remain the responsibility of the user unless a future controlled execution mode is explicitly approved and implemented with strict safeguards.

## Execution Mode Separation

The following systems must never be mixed:

### Auto-Trading

User-defined rule-based automation. It acts only according to user-selected valid conditions.

### TITAN AI Copilot Execution

User-approved setup plus TITAN AI protection/supervision logic. It may recommend or simulate protective actions when risk increases. Live execution requires explicit authorization and additional safeguards.

### TITAN AI Auto Pilot

Future autonomous research-driven system. No manual setup selection. It must remain simulation-only until validation, diagnosis, and controlled trial gates prove readiness.

## Safety Rules

This project must preserve these rules:

1. No guaranteed profit claims.
2. No guaranteed prediction accuracy claims.
3. No live exchange order placement until explicitly approved.
4. No fake live AI provider claims.
5. No invalid stop-loss or target structure should render as a normal valid signal.
6. Simulation and live execution must stay clearly separated.
7. User-owned final authority must remain visible.
8. Risk Score, Execution Readiness, Consensus Bias, and Position Allocation must remain semantically separate.
9. Backend/domain logic must not be hidden inside decorative UI code.
10. Mission Center must be the source of active mission supervision truth.
